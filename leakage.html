Financial & Inventory Leakage

១. និយមន័យ

Leakage = ការបាត់បង់ធនធាន (លុយ/ទំនិញ) ដោយ មិនមាន record និងម្ចាស់ហាង រកមូលហេតុមិនឃើញ។

វាខុសពី "ការចំណាយ" ធម្មតា — ការចំណាយយើងដឹង។ Leakage = បាត់ស្ងាត់ៗ បន្តិចម្តងៗ រហូតក្លាយជាការខាតធំ។

២. Financial Leakage

ក. Cash Skimming (លួចលុយសុទ្ធ)

  • Cashier លក់ទំនិញ ទទួលលុយ តែ មិនកត់ចូល system
  • លុយនោះចូលហោប៉ៅ cashier
  • ហាងគិតថា ទំនិញនៅ តែការពិតលក់ហើយ

ខ. Price Manipulation (កេងតម្លៃ)

  • ចុះតម្លៃឱ្យសាច់ញាតិ/មិត្ត ដោយគ្មានការអនុញ្ញាត
  • កត់តម្លៃទាបជាងពិត ហើយយកលុយលើស

គ. Refund / Void Fraud

  • ធ្វើ "void" ការលក់ក្លែងក្លាយ ហើយយកលុយ
  • Refund ឱ្យ customer បន្លំ

ឃ. Supplier Overpayment

  • បង់លុយ supplier ច្រើនជាងវិក្កយបត្រពិត
  • ទទួលទំនិញតិច តែបង់ច្រើន

៣. Inventory Leakage

ក. Theft (ការលួច)

  • Employee theft (បុគ្គលិកលួច)
  • Shoplifting (customer លួច)
  • Supplier delivery short (ដឹកមកខ្វះ)

ខ. Shrinkage (ស្តុករួញ)

  • ផុតកំណត់ (expired)
  • ខូច / បែកធ្លាក់ (damaged)
  • រលួយ (perishable goods)

គ. Administrative Error (កំហុសកត់ត្រា)

  • ទទួលស្តុកចូលខុសចំនួន
  • កត់ខុស unit (កេស vs ដប)
  • មិនបាន update system ពេលលក់
កំណត់សម្គាល់៖ តួលេខ shrinkage ជាមធ្យមនៅ retail ប្រហែល 1%–3% នៃ revenue (តួលេខអន្តរជាតិ — មិនទាន់ verify សម្រាប់កម្ពុជា)។ ហាងតូចគ្មាន system អាចខ្ពស់ជាងនេះ។

៤. ហេតុអ្វីហាងតូចកម្ពុជារង Leakage ខ្លាំង?

  • កត់ត្រាដោយដៃ (កំណត់ហេតុលើក្រដាស ឬចាំក្នុងក្បាល)
  • គ្មាន receipt
  • ម្ចាស់ហាងមិននៅ → cashier ធ្វើតាមចិត្ត
  • គ្មាន stock count ទៀងទាត់
  • គ្មាន report → មិនដឹងថាខាតត្រង់ណា

៥. Kambuja POS ដោះស្រាយយ៉ាងម៉េច (ផ្គូផ្គងម្តងមួយ)

Leakage Point POS Mechanism លទ្ធផល
Cash skimming រាល់ sale កត់ក្នុង DB + receipt ពិបាកលួចលុយ
លក់មិនកាត់ស្តុក Sale → auto stock decrease Stock ត្រូវនឹងការពិត
ស្តុកបាត់មិនដឹង Stock Movement + Adjustment log ឃើញចលនាគ្រប់ items
Price manipulation តម្លៃ lock ដោយ Admin Cashier ប្តូរមិនបាន
Void/refund fraud Audit log គ្រប់ action តាមដានបាន
មិនដឹងថាខាត Reports ពី real data ឃើញ profit/loss ច្បាស់
នរណាធ្វើអ្វី Role: Admin / Cashier ទទួលខុសត្រូវច្បាស់

៦. POS Workflow ការពារ Leakage

  1. Admin បង្កើត product (តម្លៃ lock)
  2. Admin ទទួលស្តុក → stock កើន (កត់ត្រា)
  3. Cashier លក់ → stock ថយ auto + receipt
  4. ប្រព័ន្ធកត់រាល់ចលនា (audit)
  5. Admin ធ្វើ stock count → Adjustment បើខុស
  6. Reports បង្ហាញការពិត → គ្មានកន្លែងលាក់

៧. Angle ស្រាវជ្រាវសម្រាប់ Project/Presentation

  • Problem: ហាងតូចកម្ពុជាខាតលុយ ដោយ leakage ដោយសារគ្មាន system
  • Cause: manual process, no tracking, no accountability
  • Solution: POS + Inventory (Kambuja POS) ជា case study
  • Result: real-time tracking, audit, accurate reports
  • Demo: បង្ហាញ system ពិត
